# 战略咨询常用框架 cheatsheet

> 模块 02 配套参考：列候选框架时按主问题类型从这里挑 1–3 个组合使用。**不限三选一，可叠加**——复杂研究通常 2–4 层叠加，每层用不同框架。

## 怎么用这份 cheatsheet

模块 02 工作流第 2 步（列候选框架）执行时——

1. 看主问题属于哪类：产业链结构 / 演进阶段 / 市场竞争 / 切入决策（多数研究跨多类）
2. 从对应组里挑框架，往主问题上对——能直接套就单选；多个角度都需要就组合 2–4 层
3. 给每层指明"输出形态"（参考 `judgment_framework_patterns.md`：公式 / 矩阵 / 决策树 / 链路）
4. 让用户在候选里选 / 调整 / 推翻

模块 02 起草框架后，必须做"领域专家 30 秒回检"——AI 凭印象拼框架时容易犯领域错误（比如把"氢电耦合发电反送电网"当氢能产业主链）。回检要点参考模块 02 工作流。

## 一、产业链结构层

回答"产业链长什么样、价值在哪一环"。

### Porter 价值链

**外观**：把产业活动拆成"主活动"（进料 → 生产 → 出料 → 销售 → 售后）+ "支持活动"（基础设施 / 人力 / 技术 / 采购）

**适用场景**：

- 把陌生产业拆成结构化的环节
- 给定产业，看价值在哪一环创造、哪一环捕获
- 制造业 / 重资产产业（电力、化工、钢铁）特别适用

**容易踩的坑**：把"价值链"和"产业链"混用。价值链是单一企业内部的活动链；产业链是跨企业的上下游。研究产业要用产业链版本（产业链 = 上游 → 中游 → 下游 + 配套）。

**输出形态**：因果链路（横向链条）+ 节点说明

### Smile Curve（微笑曲线）

**外观**：横轴是产业链各环节，纵轴是附加值。典型形态是 U 型——上游 R&D / 关键资源 + 下游品牌 / 服务高利润，中间制造低利润。

**适用场景**：

- 判断"价值集中在产业链哪一头"
- 看谁有定价权 / 议价能力
- 决定切入哪一环（避开微笑底部）

**容易踩的坑**：不是所有产业都是 U 型。有些产业是倒 U 型（中间制造毛利最高，比如部分专利垄断行业）；有些是 L 型（价值集中在一端）。先验证形态再用。

**输出形态**：因果链路 + 价值标注

### 产业生态图（Ecosystem Mapping）

**外观**：把"主链 + 支撑机构 + 监管 + 资本 + 用户"画在一张关系图上

**适用场景**：

- 产业涉及多方主体（金融 / 监管 / 标准制定 / 用户）
- 判断"本组织进入产业链需要哪些外部协同方"
- 平台型产业 / 强政策依赖产业

**容易踩的坑**：图画太花、关系连太多——失去重点。聚焦"和决策相关的关系"，无关方位省略。

**输出形态**：网络图 / 关系矩阵

## 二、产业演进层

回答"产业现在处在哪个阶段、未来怎么变"。

### 行业生命周期

**外观**：横轴时间，纵轴市场规模。四阶段：导入 / 成长 / 成熟 / 衰退。

**适用场景**：

- 判断"现在进入是早是晚"
- 不同阶段对应不同切入策略（导入期重技术 / 成长期重份额 / 成熟期重效率 / 衰退期重退出）

**容易踩的坑**：把"全行业生命周期"和"特定细分生命周期"混淆。行业整体可能在成长，但具体细分可能已成熟。

**输出形态**：阶段标签 + 当前位置

### 技术成熟度 TRL（Technology Readiness Level）

**外观**：1–9 级——TRL 1（基础研究）→ TRL 9（商业化部署）

**适用场景**：

- 技术路线评估
- 多技术对比（同一产业内的几条技术路径处于不同 TRL）
- 决定"做研发投入还是做产业化投资"

**容易踩的坑**：示范项目（TRL 7-8）数据被当作商业化（TRL 9）数据用——示范项目通常带补贴和优惠条款，成本失真。

**输出形态**：TRL 评分 + 关键里程碑

### Gartner Hype Cycle

**外观**：5 阶段——技术触发 → 期望膨胀 → 幻灭低谷 → 实质爬升 → 高原期

**适用场景**：

- 新兴技术 / 新概念产业研究
- 判断"是不是被炒过头了"
- 区分"短期热点"和"长期价值"

**容易踩的坑**：Hype Cycle 是定性的，不是预测工具。某技术是否会从"幻灭低谷"爬到"高原期"取决于多个因素，不是 Hype Cycle 模型自己能给出答案的。

**输出形态**：阶段标签 + 演进路径

## 三、市场结构与竞争层

回答"竞争格局怎么样、谁会胜出"。

### Porter 五力

**外观**：5 力——现有竞争者 / 潜在进入者 / 供给方议价 / 需求方议价 / 替代品威胁

**适用场景**：

- 判断"这个市场赚不赚钱"（盈利水平由 5 力决定）
- 判断"本组织进入面临什么阻力"
- 成熟产业研究

**容易踩的坑**：5 力分析容易写成"五个分项写一段就完事"。真正有用的是看 5 力之间的相互作用——比如供给方议价高 + 替代品威胁高 = 双重挤压。

**输出形态**：5 力评分（强 / 中 / 弱）+ 主导力解释

### PESTEL

**外观**：6 维宏观环境扫描——政治 / 经济 / 社会 / 技术 / 环境 / 法律

**适用场景**：

- 跨境业务 / 强政策依赖产业
- 中长期趋势研究
- 风险扫描

**容易踩的坑**：6 维分项写成清单流水账。真正有用的是看哪些维度对本案产生"非线性影响"——比如某政策一旦出台直接重构市场。

**输出形态**：6 维 × 影响强度矩阵

### BCG 矩阵 / GE-McKinsey 9 格

**外观**：

- BCG：市场份额 × 增长率，4 格（明星 / 现金牛 / 问号 / 瘦狗）
- GE-McKinsey：行业吸引力 × 自身实力，9 格

**适用场景**：

- 业务组合定位
- 多业务 / 多产品 / 多区域选哪个重点投入
- 切入新业务时判断"投入"还是"观察"

**容易踩的坑**：维度评分主观——容易"想要投入的业务就把维度评高"。两个维度都要有可独立量化的子指标支撑（不是拍脑袋）。

**输出形态**：矩阵 + 9 格 / 4 格定位

## 四、切入决策层

回答"本组织应该做什么、什么时候做、怎么做"。

### McKinsey Three Horizons（三层面增长）

**外观**：

- H1（0–2 年）：当前主营 / 现金牛 / 优化提效
- H2（3–5 年）：增长业务 / 已验证商业模式 / 扩大投入
- H3（5 年+）：长期期权 / 创造未来 / 早期播种

**适用场景**：

- 组织多元化业务组合规划
- 新业务进入决策（判断属于哪一档）
- 资源配比（H1 / H2 / H3 三档资源比例）

**容易踩的坑**：把所有"未来概念"都塞进 H3。H3 不是"概念性想法"，是"已经能识别的长期期权"——必须有可观察的早期信号。

**输出形态**：三档分类 + 资源配比建议

### Make-Buy-Partner 决策树

**外观**：自建 / 收购 / 合作三选一的决策分支——按"核心能力 vs 非核心 / 时间紧迫度 / 资本可承受度 / 文化适配度"分支判断

**适用场景**：

- 进入新业务的形态选择
- 跨界 / 不熟悉领域的最优进入路径
- 绑定上下游伙伴的策略

**容易踩的坑**：默认"自建是大型组织首选"——但跨界自建失败案例不少，应该认真比较 Buy 和 Partner 的真实成本。

**输出形态**：决策树 + 推荐路径 + 备选

### Ansoff 矩阵

**外观**：现有产品 / 新产品 × 现有市场 / 新市场，4 格——市场渗透 / 市场开发 / 产品开发 / 多元化

**适用场景**：

- 增长策略选择
- 区分"扩张"和"转型"

**容易踩的坑**：多元化（新产品 + 新市场）是风险最高的格——但常被乐观估计。多元化决策应该有更高证据门槛。

**输出形态**：4 格定位 + 推荐策略

### Real Options 思维

**外观**：把战略投资看作期权——小投入买未来选择权，大投入只在关键节点做

**适用场景**：

- 高度不确定的新兴产业（氢能 / 新能源 / 新技术）
- 阶段性投资决策
- 替代"一次性大投入"的传统投资思维

**容易踩的坑**：把"小步快跑"等同于"反复试错"。Real Options 强调"在关键节点测试假设"，不是"无目的多点撒网"。每个小投入都应该是为了验证某个具体假设。

**输出形态**：决策节点树 + 触发条件

## 多框架叠加的常见组合

针对几类典型研究，给出经过实践验证的层叠组合参考——

### 产业切入研究（如氢能 / 储能 / 新能源）

四层叠加：

- Layer 1：Porter 价值链 + Smile Curve（产业链结构 + 价值分布）
- Layer 2：行业生命周期 + TRL（演进阶段评估）
- Layer 3：GE-McKinsey 9 格（行业吸引力 × 自身实力）
- Layer 4：McKinsey Three Horizons（时间维度 + 资源配比）

### 商业可持续性研究（如新业态可行性）

三层叠加：

- Layer 1：Porter 五力 + 单位经济模型（市场结构 + 经济性）
- Layer 2：行业生命周期（演进阶段）
- Layer 3：判断公式（紧凑形态：成立 = X × Y × Z × ...）

### 政策研究 / 监管跟踪

两层叠加：

- Layer 1：PESTEL（监管语境）
- Layer 2：演变方向矩阵（多版本 × 演变方向）

### 技术路线评估

三层叠加：

- Layer 1：TRL（技术成熟度评估）
- Layer 2：决策树（多节点路径）
- Layer 3：Real Options（阶段性投资）

### 跨界 / 多元化研究

四层叠加：

- Layer 1：Ansoff 矩阵（定位扩张方向）
- Layer 2：GE-McKinsey 9 格（吸引力 + 实力）
- Layer 3：Make-Buy-Partner（进入形态）
- Layer 4：McKinsey Three Horizons（时间维度）

## 关键提醒

**1. 框架是为问题服务的，不是反过来**

无现成框架贴合主问题时，按主问题"要回答什么"自己搭原创框架。框架的价值是把研究员的判断结构化，框架本身没有神圣性。

**2. 跨框架兼容性**

不同框架的维度可能重叠（GE 9 格的"行业吸引力"和 Porter 五力的"市场盈利能力"重叠）。叠加时去重，避免重复评估同一件事。

**3. 框架的"容易踩的坑"比"适用场景"更重要**

每个框架上面都列了"容易踩的坑"——这些是实际使用中最容易出错的地方，比"什么时候用"更值得反复回看。

**4. AI 不替你做的**

按这份 cheatsheet 选框架时，AI 能做的是建议候选 + 解释适用场景。**框架是否真的回答主问题、维度组合是否合理、领域知识是否对得上**——研究员必须接手判断。模块 02 的"领域专家 30 秒回检"小节专门处理这一关。
