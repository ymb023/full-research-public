# 本地资料处理协议

> 模块 04 证据矩阵 + 模块 01 边界声明的**配套必读**。本协议是本地资料处理的唯一权威——其他模块引用本协议，不重复定义。
>
> **本协议与 `evidence_sourcing_protocol.md` 是配套关系**：本地资料是证据来源的最高优先级，联网采集协议在本地资料消化完成后才启动。

---

## 铁律

```
NO LOCAL MATERIAL WITHOUT RECOGNITION-LEVEL LABEL
```

任何本地资料引用为证据时，必须显式标注"识别等级"（5 档之一）。**只有"完整识别"等级的证据可以直接进入证据矩阵正文**；其他四档进"待研究员复核"清单。

违反铁律的证据将在 模块 05 反方审稿被剔除，相关结论强制降档。

---

## 一、核心原则

研究员手头的本地资料（政策原文、内部报告、数据表、之前研究产出）通常是**一手或半一手**的，质量高于联网二手报道。证据采集的来源优先级是：

```
本地一手（政策原文 / 内部报告 / 合同样本 / 内部数据）
        ↓ 最高优先级
联网一手（部委官网 / 上市公司披露 / 原始统计）
        ↓
联网二手（媒体报道 / 咨询机构报告 / 行业自媒体）
        ↓
AI 既有知识（仅成熟概念解释，不涉具体数字）
```

**模块 04 证据采集必须按此顺序穷尽——本地资料没消化完之前，不启动联网搜索**。这一条由 `evidence_sourcing_protocol.md` 的铁律 `LOCAL FIRST-HAND BEFORE WEB SEARCH` 强制。

---

## 二、资料形态分类与 AI 识别能力边界

不同形态的资料，AI 识别准确性差异极大。研究员注入资料时**必须做形态分类**，AI 据此调用不同工具并给出不同识别等级：

| 资料形态 | AI 能可靠识别 | 容易识别错的部分 | 几乎无法识别 | 建议工具 |
|---|---|---|---|---|
| **文本型 PDF**（政策原文/报告） | 正文文字 | 多列布局 / 表格线性化 | 含手写注释 | `anthropic-skills:pdf` |
| **扫描型 PDF**（老文件/内部文件） | 简单印刷体 OCR | 数字识别（0/O、3/8）/ 法规条款编号 | 模糊 / 倾斜扫描 | `anthropic-skills:pdf`（OCR 模式） |
| **含表格 PDF** | 单元格文字 | 跨页表格 / 合并单元格 | 嵌入图表数据 | `anthropic-skills:pdf` |
| **含图表 PDF** | 图表标题 | 坐标轴数值 | **图表内具体数据** | 研究员人工摘录 |
| **Word 文档** | 正文 | 域代码 / 引用 / 修订痕迹 | 嵌入对象 / 公式 | `anthropic-skills:docx` |
| **Excel** | 单 sheet 简单数据 | 公式 / 隐藏行列 / 数据透视 | 多 sheet 引用 / 图表 | `anthropic-skills:xlsx` |
| **图片**（剪报/PPT 截图） | 大字标题 | 小字说明 | 图表数据 | 研究员人工摘录 |

**核心问题**：研究员最需要的"关键数字、条款编号、表格数据"恰好集中在 AI 识别**最容易出错**的位置。本协议靠"识别等级"标签把这种不准确性显式化，避免 AI 误读伪装成"已验证"。

## 三、识别等级 5 档（Recognition Levels）

每条本地资料证据**必须**打识别等级标签：

| 等级 | 定义 | 可否进证据矩阵正文 |
|---|---|---|
| **完整识别** | 文本型 PDF / Word 直读，无表格无图表，AI 提取无歧义 | ✅ 可进 |
| **部分识别** | 含表格被线性化 / 图表丢失 / 多列重排 | ❌ 不可进，进"待复核" |
| **仅文字识别** | 扫描件 OCR / 图表数据全丢 | ❌ 不可进，进"待复核" |
| **摘要识别** | AI 自己总结的、非原文摘录 | ❌ 不可进，进"待复核" |
| **未识别** | 加密 / 模糊 / 嵌入对象 / 工具失败 | ❌ 不可进，提示研究员人工处理 |

研究员对"待复核"条目做抽样回原文核对后，可以**手动升档为"完整识别"**——前提是研究员显式声明"已核对原文"。

## 四、资料登记格式

### 何时登记

- **模块 01 边界声明阶段**：默认登记点。研究员注入主体资料
- **任何模块**：可动态追加。研究员在研究中途想到补充资料时随时加

### 登记表结构

每份本地资料登记如下字段：

| 字段 | 说明 | 示例 |
|---|---|---|
| **资料标题** | 文件原名或描述 | 发改能源〔2025〕650 号文 |
| **形态** | 见第二节分类 | 文本型 PDF |
| **路径** | 本地文件路径 / 文本来源 | `D:/绿电直连/政策/650号文.pdf` |
| **关联子判断** | 来自 模块 02 / 03 | A1 / B3 |
| **初步识别等级** | 5 档之一 | 完整识别 |
| **建议工具** | 见第二节工具列 | `anthropic-skills:pdf` |
| **备注** | 任何特殊说明（手写注释 / 含敏感信息脱敏 / 多版本中的哪一版等） | —— |

### 登记时的研究员责任

研究员注入资料时**必须做**：

- 形态分类（不分类 → AI 默认按"部分识别"处理）
- 路径准确（路径错 → 工具读不到 → 整批资料标"未识别"）
- 关联子判断标注（让 AI 知道这份资料对哪个判断有用）

## 五、12 字段对本地资料的填法

证据矩阵的 12 字段（升级自 11 字段，加"识别等级"），对本地资料按以下方式填：

### 信源溯源 7 项（含新增"识别等级"）

| 字段 | 本地资料填法 |
|---|---|
| 信源机构 | 资料的发布方（政策文件填"国家发改委"；内部报告填"本机构研究部门"等） |
| 发布日期 | 资料发布日期（内部资料如有版本日期填版本日期，否则填收到日期） |
| **URL** | 本地路径，格式：`local://绝对路径 @ 文件 hash 简码`，例：`local://D:/绿电直连/政策/650号文.pdf @ sha256:abc123` |
| **访问日期** | AI 实际读取本地文件的日期 |
| **证据等级** | 一手 / 二手 / 三手——**与识别等级独立**。例：政策原文是"一手"，但若是扫描件 OCR，识别等级是"仅文字识别" |
| **识别等级**（新） | 5 档之一（见第三节） |
| **原文摘录** | 直接引用 AI 提取的关键段落（不超过 100 字，加引号）。**摘录前后必须标注"AI 在文件第 X 页 / 第 Y 段提取"** |

### 判断映射 5 项（不变）

对应假设 / 待验证事实 / 能支撑什么 / 不能支撑什么 / 缺口

## 六、工具链调用规范

### 强制调用

AI 在 模块 04 消化本地资料时，遇到以下文件**必须主动调用对应工具**：

| 文件类型 | 必调工具 |
|---|---|
| `.pdf`（含 PDF 扩展名） | `anthropic-skills:pdf` |
| `.docx` / `.doc` | `anthropic-skills:docx` |
| `.xlsx` / `.xls` / `.csv` | `anthropic-skills:xlsx` |
| `.txt` / `.md` / 直接粘贴文本 | 直接 Read 工具 |
| `.pptx` | `anthropic-skills:pptx` |
| 图片（`.png` / `.jpg`） | 视觉理解（Claude 多模态） |

**禁止跳过工具直接凭文件名/路径"猜测"内容**——AI 只能基于工具实际读取的内容填证据矩阵。工具未启用或读取失败时，识别等级填"未识别"，提示研究员人工处理。

### 工具失败兜底

工具调用失败（文件加密 / 模糊 / 复杂版式无法解析）时：

1. 标识别等级"未识别"
2. 提示研究员："文件 X 工具读取失败，建议人工摘录关键内容并粘贴"
3. **禁止假装读到了内容**——这是 v3 反幻觉协议的延续

## 七、责任分工：AI 提取 vs 研究员吃透

| 动作 | AI 责任 | 研究员责任 |
|---|---|---|
| 文件读取 | 调用工具 + 标识别等级 + 写"AI 在第 X 页 / 第 Y 段提取" | —— |
| 关键数字摘录 | 提取 + 标注精度 | **抽样回原文核对**（推荐 20% 抽样） |
| 政策条款解读 | 摘录条款原文 | 必须读全文，理解"应""可""鼓励"等不同强度 |
| 财务口径 | 摘录数字 | 区分"营业收入""营业总收入""主营业务收入"等不同口径 |
| 措辞分寸 | 摘录原文 | **吃透不同政治分寸的措辞**（体制内正式汇报场景 AI 不分） |

### 一个研究员不能外包的事

**关键数字、条款编号必须研究员亲自回原文核对**。AI 误读 PDF 比联网搜索幻觉更隐蔽——研究员可能以为是原文出处，实际是 AI 误读的版本。

## 九、典型反例

| 反例 | 为什么不行 | 修正 |
|---|---|---|
| AI 读 PDF 后直接给"大约 30%-40%" | "大约"+无识别等级 | 必须标识别等级 + 给精确数字 + 标位置（第 X 页） |
| 扫描件 OCR 后当"完整识别"用 | OCR 识别等级最高只能是"仅文字识别" | 降档为"仅文字识别"，进"待复核" |
| Excel 多 sheet 数据，AI 只读了第一个 sheet 就给结论 | 部分识别，识别等级标错 | 标"部分识别"，提示研究员"还有 X 个 sheet 未读" |
| 图表数据 AI 看不到，但凭图表标题"推测" | 推测 = 编造 | 识别等级"未识别"，提示研究员人工摘录 |
| Word 修订模式的稿子，AI 读出修订前的版本当最终版 | 版本错位 | 必须标"修订模式 → 已读修订前/后版"，研究员确认是哪个版本 |
