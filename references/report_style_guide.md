# 成熟研究报告体例指南

用户提出"成稿"、"出报告"、"成熟报告体例"、"按示例风格"、"给领导看的报告"、"机构专题报告"、"简报"等要求，或六步法产出需要转成最终报告时，引用本指南。

## 核心原则

最终报告不是研究过程的逐字记录。隐藏脚手架。读者应该看到：什么判断重要，为什么有支撑，对服务对象意味着什么，下一步做什么。

不要把"问题定义"、"判断框架"、"假设清单"、"证据矩阵"、"反方审稿"、"建议分层"作为最终报告标题——除非用户明确要求加方法论附录。

## 两类报告外壳

| 外壳 | 何时用 | 开头结构 | 收尾结构 |
|---|---|---|---|
| 领导汇报式分析报告 | 报告面向领导或某个决策群体 | 标题；报送对象；一句研究缘起，以"报告如下："结尾 | "下一步建议"；部门；日期；可选附件 |
| 机构专题报告 | 作为机构正式报告或知识产品 | 标题；报告期号/日期行；`【核心观点摘要】`；`【正文】` | "相关建议"；批准 / 校核 / 编写；附件 |

不确定时，对外的研究知识产品默认用"机构专题报告"外壳，对内的决策备忘录默认用"领导汇报式分析报告"外壳。**两个外壳都是通用示例——用户单位有自己的报告体例时，优先用用户提供的样例。**

## 成熟报告主干结构

除非主题明显需要别的顺序，按下面这条主干写：

1. **核心判断前置**：2-4 段或要点，说出主要答案和含义
2. **形势 / 趋势 / 机制**：说明让这一议题重要的外部语境或机制
3. **证据支撑的分析**：按判断维度组织，不按资料收集顺序堆叠
4. **服务对象关联**：把外部结论映射到服务对象的资产、战略、风险、能力、机会
5. **建议 / 下一步**：用"一是、二是、三是"动作语言，按证据强度校准
6. **附件**：详细表格、数据清单、案例对照、方法论脚手架放这里

## 段落写法

每段分析性段落都应套用：

```text
[判断句]。[关键证据/数据/案例]。[对服务对象或下一判断的意义]。
```

好开头示例：

- "新能源发电成本与电价持续下降，参与电力市场收益面临挑战。"（能源）
- "某新业态因资产规模可控、收益结构清晰、运营数据稳定，成为最早进入试点的方向。"（产业）
- "这项改革试点的关键变量不在政策力度，而在地方配套能力。"（政策研究）

避免用资料叙述开头："根据材料显示"、"我们收集了"、"在证据矩阵中"、"从框架来看"。资料支撑段落，资料不是段落的主语。

## 标题层级纪律

- 一级标题：`一、二、三、四、五、六、七、`
- 二级标题：`（一）（二）（三）`
- 建议条目：`一是、二是、三是、四是、五是`
- 图表标号：`图1-1`、`表1`、`附件1`
- 避免在中文报告里出现"Background"、"Analysis"、"Recommendation"这种英文风格标题

## 核心观点摘要规则

机构专题报告必须在 `【正文】` 前加 `【核心观点摘要】`。

写 3-6 段紧凑段落：

- 第 1 段：核心外部趋势或机制
- 第 2 段：与服务对象的相关性、风险或资产关联
- 第 3 段：校准过的建议
- 可选段：重要的地理 / 场景 / 市场 / 政策 / 案例区分

摘要应能独立读懂。不要出现方法论标签。

## 建议措辞风格

最终建议要写成机构能用的动作：

- "建议加快研究制定..."
- "建议持续优化..."
- "建议完善..."
- "建议稳妥探索..."
- "建议积极参与..."

证据档位翻译为措辞：

| 证据档位 | 最终报告措辞 |
|---|---|
| 观察类 | "持续跟踪"、"建立监测口径"、"保持前瞻性关注" |
| 研究类 | "开展专题研究"、"摸排资源"、"形成评估框架"、"建立工作机制" |
| 行动类 | "启动试点"、"加快布局"、"推动实施"、"完善管理机制" |

不要把"观察类 / 研究类 / 行动类"作为最终报告标题，除非用户明确要求做监测表。

## 证据与数据风格

- 关键判断处优先用精确数字、时间窗口、信源名称
- 数据要放在它支撑的判断附近
- 未与原文核对过的数据，标"待核"
- 必要时把证据边界融进散文，而不是单列一节方法说明
- 密集数据、场站清单、多列对照放进附件

## 服务对象关联映射

写建议前，先用至少一个"服务对象关联"视角翻译外部发现（服务对象 = 你在研究主体声明里写的主要受众）：

- 战略：是否对齐服务对象的战略方向 / 职责重点 / 既定规划
- 资产：现有项目、资源质量、区域、技术、运营历史、内部数据可得性
- 能力：谁能执行、哪个部门或单位负责、需要什么支持
- 风险：合规、市场、政策、舆情、财务敞口（按你所在领域的风险类型展开）
- 时机：近期行动、研究储备、长期观察

这层映射如果弱，行动类建议要降档为研究类或观察类。

## 定稿前自检

提交成熟报告草稿前，逐项检查：

- 开头是否先给答案再讲过程？
- 标题是否是读者语言而不是方法论语言？
- 每个主要章节是否以判断句开头？
- 建议是否写成服务对象能采取的动作，而不是泛泛的行业建议？
- 证据边界是否体现在措辞强度里？
- 详细表格和脚手架是否进了附件？
