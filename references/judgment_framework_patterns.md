# 判断框架的形态空间

> 模块 02 配套参考：判断框架不只有"乘数公式"一种形态。研究类型决定形态。

## v6 核心提醒（先读这一段）

**复合形态不是"高级"形态**——能用单一形态解决的问题，**禁止**用复合形态。

默认从简单到复杂的选择顺序：

```
对比矩阵 > 单一其他形态 > 复合形态
```

理由：
- 复合形态会让后续证据采集量翻倍，研究员负担增加
- 复合形态容易掩盖单一框架已经够用的事实
- "决策树 + 乘数公式" 不是默认组合——这是 AI 的偏好，不是研究本身的需求
- 任何形态如果说不清"我为什么需要它"，就不要用

模块 02（框架收敛）的目的是把主问题压成一个研究员能用的判断骨架。骨架的具体形态取决于研究类型——选错形态会让后续证据收集走偏。下面列出四种常见形态，按适用场景排序。

## 形态一：乘数公式（必要条件相乘）

**外观**：

> [主问题] 成立 = [因素 1] × [因素 2] × ... × [因素 N]

**核心特性**：每个因素都是"必要条件"——任何一个为零，整体就不成立。

**适用场景**：

- 商业可持续性判断（"这个商业模式成立吗"）
- 投资可行性判断（"这个项目能赚钱吗"）
- 系统正常运转判断（"这套系统能跑起来吗"）

**典型例子**：Token 出海案例的"商业链条成立 = 需求 × 付费 × 单位经济 × 定价权 × 部署可扩张性"。

**容易踩的坑**：把"加分项"误塞进乘数。乘数公式只接受必要条件——一个因素若仅是加分项（缺它判断仍可成立），应挪到公式外作为修饰说明。

---

## 形态二：对比矩阵（多维度归类）

**外观**：

```
              方向 A    方向 B    方向 C
维度 1         [...]    [...]    [...]
维度 2         [...]    [...]    [...]
维度 3         [...]    [...]    [...]
```

**核心特性**：把每个观察单元归类到一个象限，整体判断从矩阵的分布形态读出。

**适用场景**：

- 政策演变方向判断（"政策朝哪几个方向走"）
- 多版本 / 多区域 / 多主体对比研究
- 产业格局演化判断

**典型例子**：广东电力市场规则演变案例的"规则模块 × 演变方向"矩阵。

**容易踩的坑**：维度选取偏到一边（比如全是"市场化"维度，缺了"风险管控"维度），导致一类演变被系统性低估。

---

## 形态三：决策树（条件分支）

**外观**：

```
判断 X 成立吗？
├── 节点 1 满足 → 进入节点 2
│   ├── 节点 2 满足 → ...
│   └── 节点 2 不满足 → 终止：X 不成立
└── 节点 1 不满足 → 终止：X 不成立
```

**核心特性**：每个节点是独立判断，前一节点决定是否进入下一节点。任何一节点不成立可以提前终止，省下后续证据收集成本。

**适用场景**：

- 技术路线评估（"这条路线在我们这能跑吗"）
- 多阶段决策（"先做 A 还是先做 B"）
- 路径依赖型判断（"前提条件是否满足"）

**典型例子**：氢能制取技术路线评估的多节点决策树。

**容易踩的坑**：把并存路线写成互斥分支。很多技术路线 / 业务路径其实可以并存，决策树会过早收敛到单一路线。

---

## 形态四：因果链路（追溯式判断）

**外观**：

```
[起点观察] → [中间环节 1] → [中间环节 2] → ... → [终点判断]
```

**核心特性**：每一环都需要被证据独立支撑，链条任何一处断裂整条结论就不成立。

**适用场景**：

- 影响传导研究（"X 对 Y 有什么影响"）
- 落地映射（"行业现象 → 服务对象动作"——这是模块 06 默认形态）
- 归因分析（"Y 为什么发生"）

**典型例子**：Token 出海案例第六步的落地映射链（Token 调用 → 算力 → 耗电 → 新型电力负荷 → 服务对象承接）。

**容易踩的坑**：链条中间环节的"为什么这一环成立"没说清楚，AI 推理被当成证据用。

---

## 怎么选形态

按主问题类型选：

| 主问题类型 | 推荐形态 |
|---|---|
| "X 是否成立 / 可持续 / 真实"（多必要条件并存） | 乘数公式 |
| "X 演变方向是什么 / X 之间有什么差异" | 对比矩阵 |
| "X 在我们的条件下是否可行 / 选哪条路" | 决策树 |
| "X 对 Y 有什么影响 / Y 由什么导致" | 因果链路 |

不确定时可以先用对比矩阵——对比矩阵是最不容易出错的形态，研究做到一半发现别的形态更合适，再迁移过去。

## 混合形态

复杂研究往往是混合形态。比如：

- 主问题用乘数公式判断"是否成立"
- 每个乘数内部用决策树判断"在我们条件下成立的路径"
- 多场景比较时用对比矩阵
- 最后映射到落地对象用因果链路

混合不是问题，关键是研究员能清楚说出"这一层用什么形态、为什么用这个形态"。
