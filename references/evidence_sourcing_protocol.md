# 证据采集协议

> 模块 04 证据矩阵的**前置必读**。本协议主管**联网采集**部分；本地资料处理见 `local_materials_protocol.md`（配套必读）。
>
> **违反本协议的证据将在 模块 05 反方审稿中被剔除，相关结论降档。**

---

## 铁律

```
LOCAL FIRST-HAND BEFORE WEB SEARCH
```

证据来源优先级是**强制**的：

```
本地一手（政策原文 / 内部报告 / 合同样本）
        ↓ 最高优先级
联网一手（部委官网 / 上市公司披露 / 原始统计）
        ↓
联网二手（媒体报道 / 咨询机构报告 / 行业自媒体）
        ↓
AI 既有知识（仅成熟概念解释，不涉具体数字）
```

**本地资料没消化完之前，不启动联网搜索**。本地资料的处理协议见 `local_materials_protocol.md`。

---

## 一、核心原则

AI 在 模块 04 的工作不是"基于既有知识填证据矩阵"，而是 **"先消化本地 → 边界自检 → 联网补缺 → 填 12 字段"**。本协议主管后三步；第一步由 `local_materials_protocol.md` 主管。

执行顺序：

```
消化本地资料（local_materials_protocol.md）
        ↓
对照模块 01 边界自检
        ↓
（落在排除范围 → 直接拒绝）
        ↓
按工具顺序联网采集（缺什么补什么）
        ↓
（联网搜索 → 取原文 → 询问研究员）
        ↓
按 12 字段格式填入证据矩阵
        ↓
（缺任一项 → 视为"未验证"，进"待补"清单）
```

---

## 二、第一步：边界自检（先于一切搜索）

每条证据采集**之前**，先对照 模块 01 的"边界声明"做自检：

- 证据涉及的内容是否落在"覆盖范围"内？
- 证据涉及的内容是否落在"不回答什么"的排除范围？

判定结果：

| 情况 | 处理 |
|---|---|
| 落在覆盖范围 | 进入第三节"联网采集" |
| 落在排除范围 | **直接拒绝**——不要心存侥幸"也许有点参考价值" |
| 有歧义 | 暂时填入但显式标注 `[待边界确认]`，提醒研究员裁决 |

### 真实教训（绿电直连研究案例）

| 模块 01 边界声明 | 错误引用 | 正确做法 |
|---|---|---|
| 排除"海外市场" | 引用麦肯锡国际 PPA 数据支撑中国市场结论 | 拒绝填入，改为"中国市场数据待补" |
| 排除"绿证、标准化绿电交易" | 引用绿证交易量作为直连市场证据 | 拒绝填入 |
| 时间窗 2026-2030 | 引用 2018 年市场数据外推到 2030 | 标注"数据时效不匹配"，降档使用 |

**边界自检放在最先做，是因为它是最便宜的过滤器**——还没花时间搜索，就先把不该填的证据挡在门外。错误若漏到 模块 05 反方审稿才发现，代价大几倍。

---

## 三、第二步：联网采集（工具调用顺序，不能跳步）

通过边界自检的证据需求，按以下顺序采集：

### (a) 判断是否必须联网

**必须联网查**——只要证据涉及以下任一类型：

| 类型 | 例子 |
|---|---|
| 数字与统计 | 市场规模、增长率、占比、装机量、收入数据、毛利率 |
| 政策法规 | 部委文件、地方规定、行业标准、监管要求 |
| 价格与费率 | 电价、产品价格、关税、补贴标准、合同价格 |
| 近期事件 | 12 个月内的并购、签约、政策出台、人事变动、领导讲话 |
| 领导人 / 机构 | 在任领导、机构设置、职务变动、组织架构 |
| 公司具体数据 | 产能、营收、客户清单、合同金额、项目清单 |
| 标准与口径 | 统计口径、行业标准、术语定义、专业术语 |

**可凭既有知识**——仅以下类型：

- 成熟概念的解释（什么是 PPA、什么是 IRR、什么是源网荷储）
- 公认的历史背景（不涉及具体数字时）
- 普适的方法论原则（不涉及最新实践时）

但即使这些，研究员追问"出处"时必须立刻补查或承认"无出处"。

### (b) 工具调用顺序

1. **联网搜索（多源验证）**——用 1-3 个不同关键词组合搜，看是否有 2-3 条来源给出一致事实
2. **取原文**——对关键政策文件、统计数据、公司披露，必须取原文做精确引用，不能只看搜索摘要
3. **询问研究员**——前两步都查不到时，明确说"找不到，建议研究员从内部数据 / 行业访谈 / 付费数据库补充"
4. **绝对禁止**——跳过 1-3 直接凭记忆填表，或编造"看似合理的数字"

### (c) 工具不可用时

若联网搜索在当前会话中不可用，AI **必须显式告诉研究员**："本次未联网查证"，把所有未查证证据行标 `[未查证·待补]`。**禁止假装查过**。

## 四、第三步：按 12 字段格式填表

每条证据必须填以下 **12 个字段**。**任一字段为空 → 该行视为"未验证"，只能进"待补"清单，不能进入证据矩阵正文**：

### 信源溯源类（7 项 · 协议强制）

| 字段 | 说明 | 联网证据示例 | 本地资料示例 |
|---|---|---|---|
| **信源机构** | 发布方全称（不简写） | 国家发展改革委 | 本机构研究部门 |
| **发布日期** | 文件 / 报道发布的精确日期 | 2025-05-30 | 2025-12 |
| **URL** | 可点开的原始链接 / 本地路径 | https://www.ndrc.gov.cn/... | `local://D:/绿电直连/政策/650号文.pdf @ sha256:abc123` |
| **访问日期** | AI 实际查到/读到的日期 | 2026-05-06 | 2026-05-06 |
| **证据等级** | 一手 / 二手 / 三手 / AI 二次概括 | 一手 | 一手 |
| **识别等级**（v5 新增） | 5 档之一（仅本地资料）：完整识别 / 部分识别 / 仅文字识别 / 摘要识别 / 未识别。联网证据填"N/A"或"完整识别" | N/A | 完整识别 |
| **原文摘录** | 关键段落原文（不超过 100 字，加引号）。本地资料须配"AI 在第 X 页 / 第 Y 段提取" | "应按国务院价格..." | "新能源直连项目应配套建设储能..."（第 3 页第 2 段） |

**关于"识别等级"**：仅本地资料的"识别等级"可能 ≠ "完整识别"。**只有"完整识别"等级的本地资料证据可以直接进入证据矩阵正文**；其他四档进"待复核"清单。详见 `local_materials_protocol.md`。

### 判断映射类（5 项 · 模块 04 判断动作）

| 字段 | 说明 |
|---|---|
| 对应假设 | 来自模块 03 |
| 待验证事实 | 具体到一个可被回答的事实点 |
| **能支撑什么** | 这条证据能直接说明的判断 |
| **不能支撑什么** | 这条证据明确不覆盖的边界 |
| 缺口 | 还需要补什么证据 |

## 五、典型反例（什么不能写进证据矩阵）

| 反例 | 为什么不行 | 修正 |
|---|---|---|
| "2030 年市场规模约 500 TWh" | 凭印象的预测，无信源 | 必须查行业研报或机构预测，给出明确出处和发布日期 |
| "据估计，毛利率约 30-40%" | "据估计"是 AI 模糊措辞 | 查招股书 / 财报原文，给精确数字 + 报告页码 |
| "业内普遍认为..." | 无具体出处 | 删除，或改为"X 机构 2025 年报告认为..." |
| "大约在 70 亿到 100 亿之间" | 大约 + 无信源 | 查具体数据；查不到则显式注明"无公开数据，建议研究员补查" |
| "据多家媒体报道" | 模糊聚合，等同无源 | 必须列出具体媒体名 + 报道日期 + URL |
| 数字未带访问日期 | 数字是动态的 | 任何数字必须配"截至 X 日，根据 Y 信源" |

## 六、与反方审稿（模块 05）的衔接

模块 05 会主动检查证据矩阵中是否有：

- 12 字段不全的行 → **必改**
- 本地资料行识别等级 ≠ "完整识别" 但被当成"已验证"用 → **必改**
- 跳过联网搜索直接填的行 → **必改**
- 边界违反的行（违反 模块 01 边界声明）→ **必改**
- 单一信源支撑的关键判断 → **建议改**
- 二手 / 三手证据被当一手用 → **建议改**

**协议执行得越严，模块 05 能挑出的问题越少。** 它是 模块 04 的纪律，也是 模块 05 的预防针。

---

## 七、研究员的不可外包责任

本协议把 AI 的责任做了硬化（必查、必引、必标边界），但研究员的责任不能外包：

- AI 查到的数据，研究员必须做 **抽样回原文核对**——AI 二次概括的"大约 30%-40%"和原文披露的精确数字（76.1% → -0.4%）可能差几个数量级。具体例子见 `cases/case_token_outbound.md` 第四步
- 关键政策文件的解读，研究员必须读全文。AI 提取的"原文摘录"可能漏掉关键限定词
- 证据等级的最终判断，研究员定。同一份券商研报，在某课题里是一手观点，在另一课题里只能算二手转引

AI 把"采集 / 标注 / 排矩阵"做扎实，研究员把"复核 / 判断 / 拍板"做扎实。两者协作，证据矩阵才有支撑结论的力量。
