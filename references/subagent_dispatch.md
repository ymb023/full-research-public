# 子代理调度详情

> SKILL.md 的"子代理模式"只保留适配矩阵（哪个模块用哪个角色）。**真正派子代理时**——主要是模块 04（事实核查员）和模块 05（审稿员）——读本文件拿调度方式、调用规约、兜底与工程纪律。

## 并行 vs 单子代理调度（v7 区分）

**并行调度**（模块 04 默认）：
- 每个核心子判断派 1 个子代理，并发执行
- 优势：上下文聚焦、多源交叉天然形成、失败隔离
- 代价：API 调用 ×N（N = 子判断数，典型 3-6）
- 调度：多个 Agent 工具调用在同一消息里发出，并行返回
- 整合：主流程汇总 N 个输出 → 合并 12 字段表 + 处理冲突 + 合并待复核 / 缺口 / 拒绝清单

**单子代理调度**（模块 05 + 模块 04 兼容模式）：
- 1 个子代理处理全部任务（如审稿员看全文）
- 优势：上下文完整、整合简单
- 用途：审稿类（需要看全局）/ 子判断数少（≤ 2）/ 成本敏感

## 调用规约

派子代理时（不论并行或单）：

1. **加载提示词模板**：从 `subagents/<role>-prompt.md` 读取角色定义
2. **打包输入契约**：按模板的"输入契约"小节，把必要内容（边界声明 / 主体声明 / 假设 / 资料）作为初始消息打包
3. **子代理启动第一动作：工具自检**（v7 新增）—— 列出自己拥有的工具，缺失立即声明
4. **子代理独立执行**：不再传当前会话的额外上下文
5. **接收结构化输出**：子代理按模板的"输出契约"返回 Markdown 报告
6. **主流程整合**：研究员审视子代理报告（并行模式下汇总 N 份），决定保留 / 修改 / 拒绝

## 兜底模式

如果当前环境没有 Agent 工具（如 codex CLI、纯提示词环境），各模块必须有"按表打勾"的兜底模式——同会话执行但严格按预设清单走，不"自由发挥"。详见各模块工作流的"兜底"小节。平台等价工具映射见 `references/platform_adaptation.md`。

## 工程要点

- 子代理看不到主会话历史 —— 提示词必须**自包含**所有必要输入
- 子代理的输出必须是**结构化**的（按预设 Markdown 模板），不是自由对话
- 子代理任务结束后**不预览下游模块**，由主流程整合
- **v7（工具自检）**：子代理必须做工具自检；并行模式下，每个子代理独立做自检并汇报
- **v7.1（硬配额）**：事实核查员子代理必须满足 6 项硬性配额（关键词组合 ≥ 5 / 联网搜索 ≥ 8 / 取原文 ≥ 3 / 原始证据 ≥ 5 / 一手来源 ≥ 2 / 英文搜索 ≥ 1）才能交付；未达配额必须做满 3 轮主题调整才能宣告"部分达成"——杜绝"一两轮就放弃"。详见 `subagents/fact-checker-prompt.md` "硬配额" 节。
- **v7.2（本地资料豁免）**：内部数据 / 政策原文 / 客户内部材料主导的课题，若本地资料已"完整识别"且满足 Q4/Q5 配额，联网部分 Q2/Q3/Q6 降为最低要求（仅做 1 轮交叉验证）。详见 `subagents/fact-checker-prompt.md` "本地资料豁免" 节。
