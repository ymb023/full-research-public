# 案例：Token 出海综合研究

> **形态**：商业可持续性判断（产业研究 / 行业判断）
> **判断框架形态**：乘数公式（成立 = 因素 1 × 因素 2 × ...）

本案例展示六步法在"判断一个新兴商业链条是否可持续"这类问题上的完整跑法。所有数据和判断均来自一次真实研究。

---

## 第一步 · 问题定义

**原始题目**：中国大模型 Token 出海发展情况研究

**展开的问题空间**（节选）：

- 信息性：哪些厂商在出海 / 部署在哪些区域 / 调用量多大
- 判断性：Token 出海是否形成可持续商业链条 / 利润是否真实 / 价值最终沉淀在哪一层
- 决策性：集团是否需要在基础设施层做承接 / 哪些区域优先

**收敛后的主问题**：

1. Token 出海是否已经从前端调用热度演化为可持续商业链条？
2. 它对集团判断 AI 新型电力负荷有什么启示？

**边界声明**：

- 委托方：集团战略研究部
- 时间范围：2024—2026
- 主体范围：中国头部大模型厂商海外业务
- 不回答什么：不替代任何单笔投资决策

---

## 第二步 · 框架收敛

**一级拆解（5 类判断）**：

1. 需求 —— 海外开发者是否真在持续使用中国模型
2. 收入 —— 调用能否转化成真实收入
3. 利润 —— Token / API 这一层能否形成稳定利润
4. 价值沉淀 —— 长期价值留在模型层还是平台/企业服务/云托管/区域节点
5. 主业影响 —— 对集团 AI 新型电力负荷判断的启示

**二级收敛（2 条主线）**：

- 主因果链：使用 → 收入 → 利润
- 约束链：价格 → 成本 → 竞争 → 合规

**三级压缩（判断公式）**：

> 商业链条成立 = 需求 × 付费 × 单位经济 × 定价权 × 部署可扩张性

**收敛路径**：AI 初版 8 章综述结构 → 收敛成 5 类判断 → 2 条主线 → 1 个公式。

---

## 第三步 · 假设生成

**子判断 1（需求层面）**：

- 正向假设：开发者层面调用量持续上升 —— 先验置信度：高
  - 需要验证：第三方平台调用排名 / 增速 / 集中度
  - 可能证伪：调用增速放缓且集中在少数 SDK 嵌入场景
- 反向假设：调用量虚高，依赖低价 —— 先验置信度：中
  - 需要验证：付费率 / 留存 / 切换成本
  - 可能证伪：付费率与平均水平相当且留存稳定

**子判断 3（利润层面）**：

- 正向假设：云端 Token 业务有较稳毛利 —— 先验置信度：低
  - 可能证伪：头部厂商招股书披露毛利率持续压缩或转负
- 反向假设：云端 Token 利润正在被快速压缩 —— 先验置信度：高
  - 可能证伪：多家披露毛利率反弹

**整体核心提问**：什么证据会推翻"商业链条成立"这个总判断？
→ 任何一个乘数（需求 / 付费 / 单位经济 / 定价权 / 部署可扩张性）长期保持低值或趋零。

---

## 第四步 · 证据矩阵

**三组关键证据**：

| 维度 | 来源 | 能支撑什么 | 不能支撑什么 |
|---|---|---|---|
| 需求侧 | OpenRouter 平台数据 | 中国模型在海外开发者平台有真实调用 / 增速明确 | 长期稳定付费 / 利润成立 / 私有化部署场景 |
| 收入侧 | 智谱招股书 + 公开信披 | 海外业务带来初步收入流 | 收入是否覆盖部署与获客成本 |
| 利润侧 | 智谱招股书云端部署毛利率 | 云端 Token / API 层利润正在快速压缩，已转负 | 所有厂商海外 Token 业务都不赚钱（单一样本） |

**回原文校对要点**（最具冲击力的一组数据）：

- AI 初版概括：「智谱云端部署毛利率大约在 30%–40% 之间」
- 回原文核对：2021 / 2022 / 2023 三年云端部署毛利率分别为 **76.1% / 5.4% / -0.4%**
- 判断含义变化：
  - 粗版 → "云端业务还有比较稳定毛利"
  - 细版 → "云端 Token / API 层利润正在快速压缩，已经转负"

这一组的差别说明：AI 给的中间结论会"差一点"，这一点点常常就是研究成不成立的分水岭。

---

## 第五步 · 反方审稿

**挑出的关键风险（节选）**：

| 等级 | 位置 | 问题 | 修改建议 |
|---|---|---|---|
| 必改 | 第 3 章利润分析 | "毛利率大约在 30%–40% 之间"与原文（76.1% → -0.4%）不符 | 改成精确数据，判断同步从"较稳毛利"调为"已转负" |
| 必改 | 第 3 章结尾 | 基于智谱单一样本得出"中国大模型云端业务全面亏损" | 改成"头部厂商已现压缩信号"，配后续交叉验证计划 |
| 建议改 | 摘要 | "Token 出海商业链条已经显著走通" | 改为"前端调用走通，商业转化和利润沉淀仍需观察" |

---

## 第六步 · 建议分层

**主业映射链**：

```
Token 出海商业链条
  → 推算调用规模 → 测算算力需求
    → 折算耗电规模 → AI 新型电力负荷判断
      → 集团基础设施承接机会
```

每一环的"为什么成立"：

- Token 调用 → 算力：模型推理算力换算系数（公开口径）
- 算力 → 耗电：单 GPU 功耗 × 集群利用率（行业 benchmark）
- 耗电 → 新型电力负荷：能源局 / 电网公司分类口径
- 新型电力负荷 → 集团承接：集团已布局的算力园区 / 区域节点

**三档建议**：

- **观察类**：建立海外 Token 调用与算力耗电监测口径
  - 监测指标：OpenRouter 中国模型月度调用占比 / 头部厂商海外业务披露毛利率 / 集团关注区域 IDC 上架率
- **研究类**：开展集团关注区域 AI 新型电力负荷专题研究
- **行动类**：本案不进此档（利润证据未交叉验证；商业链条第三环未成立）

**为什么行动类为空**：恰恰是建议分层做对的体现——证据强度不够时，主动降档或留空，比抢档显得专业。

---

## 这个案例展示了六步法的什么特性

- 判断框架收敛到一个**乘数公式**——这种形态适合"商业可持续性判断"，不适合所有研究类型（参考 `references/judgment_framework_patterns.md`）
- 关键数据回原文校对的极端例子（76.1% → -0.4% vs "30%–40%"）
- 行动类建议主动留空——证据未到行动级时不抢档
- 主业映射链是央企智库的特殊纪律，普通行业研究里这一步可能简化或省略
