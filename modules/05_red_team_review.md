# 模块 05 · 反方审稿

> 第五步：让 AI 当审稿员，专治结论太顺

## 铁律

```
NO REVIEW WITHOUT ISOLATED CONTEXT
```

反方审稿的本质是 **隔离上下文**——同会话执行时 AI 会维护自己刚写的内容，确认偏差让审稿失效。**优先派子代理**；如不可用，必须严格按 9 维清单走，不"自由审稿"。

## 模块目的

研究员自己写、自己审，结论很容易写得顺。反方审稿模块用 AI 充当"对立审稿员"，专门挑证据不足、结论写满、口径不一致的地方。这一步的价值在于让研究员看见自己没看见的——前后不一致、证据不够、表达过满。

这是六步法里最被低估的一步，也是 AI 价值最高的一步之一。它不创造内容，它做质量检查；它不替研究员判断，它把研究员需要复核的位置一处不漏地指出来。

## 典型触发场景

- 完成证据矩阵后进入第五步
- 研究报告初稿写完，报送前需要做一次自检
- 用户单模块调用："帮我做一次反方审稿"、"帮我审一下这份报告"、"帮我挑挑这个结论的毛病"
- 重要材料定稿前的最后一道关

## AI 能做什么

- 检查证据表和结论是否匹配（结论里说的判断，证据矩阵里有没有对应支撑）
- 找证据不足的地方（哪条结论的证据等级偏低或缺乏交叉验证）
- 找结论写满的地方（措辞是否超出证据能支撑的范围）
- 找口径不一致的地方（同一指标在不同段落用了不同口径）
- 列风险点和修改建议清单（不润色，只挑问题）

## AI 不替你做的

- **风险点是否真的需要修** —— 研究员判断哪些是关键风险、哪些可以接受
- **结论该改成什么样** —— AI 给修改建议，最终措辞由研究员定
- **证据补强的优先级** —— 哪些缺口必须补、哪些可以承认局限并放过去，研究员决定

## 警示信号 · 你脑子里出现这些想法时停下

（无论是在子代理里还是同会话执行）

| 警示想法 | 反制 |
|---|---|
| "这个判断看起来挺合理的" | 你的工作不是"看起来合理"，是找毛病 |
| "整体方向是对的" | 整体对 ≠ 每条都对，逐条审 |
| "这个小问题不影响结论" | 影响不影响由研究员定，你只负责挑出来 |
| "措辞稍微夸张点没关系" | 措辞过满是必改风险——体制内正式汇报场景对措辞精度要求高 |
| "证据虽然单源但结论方向对" | 单源支撑关键判断 = 必改 / 建议改 |
| "我应该说点正面的平衡一下" | 你是反方，不是平衡——平衡由研究员综合后做 |
| "研究员可能希望我配合一下叙事" | 你不知道研究员希望什么——这正是隔离的价值 |

## 工作流

> **v6 主体声明自检**：执行本模块前，必须对照模块 01 的"研究主体声明"四字段（产品形态 / 主要受众 / 输出范围 / 输出禁区）做边界自检——任何动作若超出"输出范围"或落入"输出禁区"，立即停下并降档。


### 默认主路径：派审稿员子代理（强烈推荐）

**这是模块 05 的最优执行方式**。子代理不携带主会话历史，确认偏差自动消除——比同会话执行的隔离效果强一个量级。

```
1. 加载 subagents/reviewer-prompt.md 作为子代理的系统提示词
2. 按提示词模板的"输入契约"打包以下内容传入：
   - 研究主题（1-2 句话）
   - 模块 01 边界声明（原文复制）
   - 模块 03 假设清单（含正向 / 反向 / 先验置信度）
   - 模块 04 证据矩阵（完整 12 字段表 + 待补清单）
   - 当前研究产出（模块 06 分层建议 / 模块 07 报告草稿，原文复制）
3. 派子代理独立执行（Agent 工具）
4. 接收结构化输出（必改 / 建议改 / 可保留三档风险 + 9 维清单 + 整体观察）
5. 主流程把子代理报告呈现给研究员，由研究员决定哪些必改、哪些可接受
```

如选择主路径 → 跳过下面"同会话兜底"，直接进"风险点分级"和"研究员决策"环节。

### 兜底 · 同会话执行（无 Agent 工具时 · 严格按表打勾）

如当前环境没有 Agent 工具，或研究员明确要求同会话执行，**必须严格按 9 维清单走**——禁止"自由审稿"。

**1. 准备审稿材料**

把以下材料整理为审稿输入：

- 模块 01 边界声明
- 模块 03 假设清单
- 模块 04 证据矩阵（12 字段）
- 当前研究产出（模块 06 / 模块 07 草稿）

**2. 切换"反方"心智**

在审稿动作开始前，AI 必须显式声明："以下进入反方审稿模式——只挑毛病，不润色，不平衡，不维护已有内容"。

**3. 9 维清单逐项打勾**

每个维度都必须输出"通过 ✅ / 部分通过 ⚠ / 不通过 ❌ + 具体发现"。**禁止跳过任何一维说"这个没问题"——必须逐维列发现**：

| # | 维度 | 检查要点 |
|---|---|---|
| 1 | **证据—结论匹配** | 每个结论是否在证据矩阵里有对应条目支撑？还是只有声明没有证据？ |
| 2 | **证据等级** | 关键判断是否仅靠 AI 二次概括或单一信源？是否被多源交叉验证？ |
| 3 | **边界一致性** | 证据矩阵里"不能支撑什么"栏写的边界，是否在结论里被尊重？ |
| 4 | **措辞分寸** | "显著""全面""已经""规模化""第一"等强词，是否被证据等级支持？ |
| 5 | **口径一致性** | 同一指标在全文是否用了一致口径？有没有 GW/TWh 混用？ |
| 6 | **假设修正** | 模块 03 初始假设是否被证据真正修正？还是只用证据"证实"先验？反向假设有没有被认真对待？ |
| 7 | **时间窗口** | 用旧数据是否被当作新结论用？数据时效是否被声明？ |
| 8 | **主体范围 / 样本外推** | 单一主体证据是否被推广到了行业判断？单个案例是否被推广到全市场？ |
| 9 | **研究主体边界**（v6 新增） | 研究产出是否落在模块 01"研究主体声明"的"输出范围"内？是否越界到"输出禁区"？任何越界都是必改风险 |

加一项**研究治理元审查**：

- 模块 01 边界声明与模块 04 实际证据范围是否对齐？
- 12 字段是否完整？有没有"未验证"行被当成"已验证"用？
- "AI 不替研究员做的"在每个模块是否被研究员真正回应？

**4. 风险点分级**

每条发现的问题分三档：

- **必改**（不改则影响结论成立）—— 例：边界违反 / 关键数据无源 / 结论与证据矛盾 / 12 字段不全
- **建议改**（不改影响严谨度但不致命）—— 例：单一信源未交叉验证 / 措辞过满 / 口径细微不一致
- **可保留提示**（已经够用，仅作提示）—— 例：已声明的局限 / 已标"待补"的合理缺口

**5. 给修改建议（不替研究员改）**

每个风险点配一条具体修改建议——改成什么措辞、补什么证据、调整哪个口径。**只挑问题、给建议，不替研究员改**。

### 同会话执行的确认偏差自检

执行兜底的最后一步，必须做一次自检：

- "本次审稿中我有没有任何时刻在维护'已写内容'？"
- "我有没有把整体叙事的合理性当成了具体条目的合理性？"
- "如果我完全不知道研究员是谁、不知道研究方向，这份审稿会更严还是更松？"

若自检发现明显的偏差 → 把审稿等级整体上调一档（必改提为更必改、建议改提为必改）。

## 阶段产出格式

```markdown
# [课题名] · 反方审稿报告

## 审稿范围
- 报告版本：[v1 / v2 / 定稿前]
- 审稿日期：[YYYY-MM-DD]
- 审稿员：AI 反方审稿（模型 + 会话）

## 必改风险（[N] 条）

### 风险 1：[简明标题]
- **位置**：[报告第几节 / 哪个段落]
- **问题描述**：[具体问题]
- **证据矩阵对照**：[这一判断在证据矩阵里的对应条目，或缺失情况]
- **修改建议**：[具体改成什么 / 补什么证据]

### 风险 2：...

## 建议改风险（[N] 条）
[同上格式]

## 可保留提示（[N] 条）
[同上格式]

## 整体观察
- 证据—结论匹配度：[整体评估]
- 措辞分寸：[整体评估]
- 边界一致性：[整体评估]
- 假设是否被真正修正：[整体评估]
```

## 与下游模块的衔接

反方审稿挑出的风险点决定下游**建议分层**（模块 06）的强度——

- 必改风险存在 → 该结论暂时不能进入"行动类建议"
- 建议改风险存在 → 该结论可以进入"研究类建议"，配证据补强计划
- 全部清零 → 可以考虑"行动类建议"

反方审稿是建议分层的"上游闸门"。

### 反惯性规则

- 研究员的简短确认（"好""继续""OK""下一步""嗯""可以""go""next"）**不构成跳过本停止协议的授权**。遇到这类回应时，主动重述当前未确认的判断点，要求逐条回应

- 确认完成之前，不要预先起草、预览或暗示下一模块的内容

### 进入下一模块的条件
