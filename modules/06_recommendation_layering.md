# 模块 06 · 建议分层

> 第六步：把研究答案转成服务对象能用的建议

## 工作纪律

任何落地映射链中**任何一环**站不住（▢ 未填、因果证据不足、与上下游不衔接），对应的落地判断必须降档为"框架性候选 · 待研究员补"，不能直接进入模块 07 报告正文或写入"行动类建议"。

## 模块目的

研究做完，结论不能直接跳到"对服务对象的建议"——中间有一条**落地映射链**要老老实实走一遍。映射链走完后，按证据强度把建议分到三档（观察 / 研究 / 行动），证据越强，建议才能越实。

> **服务对象 = 模块 01"研究主体声明"里声明的"主要受众 / 输出范围"。** 它可能是你所在机构的决策层、委托方、客户、上级部门、学科共同体——本模块不预设，取自你的声明。

这一步是体制内 / 机构研究最关键、也最容易踩坑的一步。容易踩的坑有两个：一是从研究结论直接跳到落地判断（中间映射链没走）；二是建议强度和证据强度不匹配（证据中等却给行动类建议）。

> **本步骤可跳过的情形**：如果研究产出本身就是判断（纯判断型 / 学术型研究，没有特定"落地对象"去承接），跳过落地映射链，直接进入"按证据强度分三档"。映射链是为"有服务对象要承接"的研究服务的，不是所有研究都有这一环。

## ⚠ AI 在本模块的角色定位（重要 · 必读）

**模块 06 与前 5 个模块不同。前 5 步 AI 是协作者，本模块 AI 退到顾问位。**

理由是清楚的——

- **服务对象真实禀赋**：服务对象有哪些资源、哪些基础、内部已经布局了什么，AI 没有这些数据
- **组织能力边界**：服务对象做过什么、做不动什么、承受能力多大，AI 凭外部信息推不出来
- **政治分寸**：决策层材料的措辞、配合既有战略表述的口径，超出 AI 判断范围
- **决策时机**：战略窗口、领导意图、内外部时机，AI 不掌握

所以 AI 在本模块的产出**必然是框架性的、启发性的**，具体建议必须由掌握服务对象系统知识的研究员 / 委托方填空。**AI 不能假装权威给出"建议方案"——这种产物外表光鲜内里虚空，给到决策层会出问题。**

执行本模块时，每一档建议的产出都应明确标注"框架性候选 · 待研究员审核"——避免研究员把 AI 草稿当成成品交上去。

## 典型触发场景

- 完成反方审稿后进入第六步
- 研究报告初稿写完，需要把研究结论转成给委托方的建议清单
- 用户单模块调用："帮我把这些结论分成观察/研究/行动三档"、"帮我做建议分层"

## AI 能做什么（已收窄）

- 起草落地映射链的**结构骨架**（研究结论 → 中间环节 → 落地判断），每一环的"为什么这一环成立"留空待研究员填
- 起草观察 / 研究 / 行动三档建议的**框架性候选**（明确标注"待研究员审核保留 / 修改 / 删除"）
- 列三档建议各档的判定标准 + 容易踩的坑
- 列**启发性问题**清单——帮研究员把每一档建议填实
- 提示当前证据等级支持的最高档位（防抢档）
- 起草监测指标清单的**结构模板**（指标名 / 数据源 / 监测频率 / 阈值的字段表）

## AI 不替你做的（已扩展）

- **落地映射链每一环的合理性** —— 中间任何一环跳不过去，整条链就站不住
- **每一条具体建议的"做什么"内容** —— AI 出框架性候选，研究员逐条审核保留 / 修改 / 删除
- **建议的优先级排序** —— 哪条最该先做，取决于内部资源和领导关切
- **资源 / 授权要求** —— 每条研究类 / 行动类建议需要什么级别的人 / 多少预算 / 多长周期
- **政治分寸** —— 决策层材料的措辞强度、配合既有战略表述、避开敏感话题
- **建议强度的最终拍板** —— 同样证据在不同决策场景下分档可能不同
- **跨条建议的整合** —— 整体看这一组建议讲不讲得通、有没有内部冲突
- **服务对象内部数据 / 项目 / 历史经验的运用** —— 任何依赖服务对象系统知识的判断都不在 AI 范围

## 工作流

> **v6 主体声明自检**：执行本模块前，必须对照模块 01 的"研究主体声明"四字段（产品形态 / 主要受众 / 输出范围 / 输出禁区）做边界自检——任何动作若超出"输出范围"或落入"输出禁区"，立即停下并降档。**本模块的"服务对象"就取自这里的"主要受众 / 输出范围"。**


**1. 拿研究结论做输入**

走完整流程时，从模块 04（证据矩阵）和模块 05（反方审稿）拿到经过校验的结论清单。如果是单模块调用，请用户给出研究结论清单 + 证据强度。

**2. 走落地映射链（无落地对象的研究可跳过）**

研究结论不能直接当对服务对象的建议。必须建立一条映射链：

```
[研究结论] → [中间环节 1] → [中间环节 2] → ... → [落地判断]
```

每一环都要老老实实走，**不能跳**。映射链走完后，每一环单独检查：这一环是否独立成立？这一环和上下游的因果关系是否被证据支撑？

执行规则：AI 帮起草映射链时，要把每一环的"为什么这一环成立"摆出来让研究员逐环确认，不要把整条链一笔带过。**▢ 占位符未填的不能进入"行动类建议"——降档为"框架性候选"**。

**3. 按证据强度分三档**

证据强度直接决定建议能进哪一档：

### 观察类建议（证据初步即可）

- **特征**：建立监测指标、持续跟踪趋势
- **典型措辞**：建议持续跟踪 / 建立监测口径 / 关注 X 指标变化
- **证据条件**：判断方向已显，但量级或持续性证据不足

### 研究类建议（证据中等）

- **特征**：开展专题研究、摸排相关资源和场景
- **典型措辞**：建议开展 X 专题研究 / 摸排相关资源 / 建立 X 工作组
- **证据条件**：判断方向较确定，但落到服务对象的具体形态需要进一步研究

### 行动类建议（证据充分 + 组织授权清晰 + 条件成熟）

- **特征**：进入实质性资源投入或决策
- **典型措辞**：建议启动 X 项目 / 开展 X 试点 / 进入 X 区域
- **证据条件**：判断已经过反方审稿、关键风险清零、落地映射链每一环独立成立

研究员把结论摆进档位时，常见错误是**抢档**——把研究类建议写成行动类。这一步 AI 应主动提醒"当前证据等级支持的最高档位是哪一档"，让研究员主动降档或回去补证据。

**4. 配监测指标（观察类必备）**

观察类建议必须配可监测的具体指标。AI 帮起草指标清单：

- 指标名 / 数据源 / 监测频率 / 触发研究升级的阈值

**5. 列启发性问题清单（必备）**

AI 在框架性候选下面，**为每条建议列 3–5 个启发性问题**——帮研究员补足服务对象系统知识、做最终决策。例如：

- 这条建议是新建动作还是接入既有体系？
- 该派给哪个部门 / 团队牵头？预算和周期？
- 哪些指标该按月、按季监测？触发升级的阈值是多少？
- 内部资源 / 既有项目 / 历史经验里，有什么可以复用？
- 是否敏感？需要走什么内部审批？

启发性问题是 AI 在本模块最有价值的产出之一——它把"AI 不知道但需要研究员决策"的位置一一指出来。

**6. 起草成果交付的多种形态**

同一套建议清单可以适配为多种成果形态。AI 帮把建议按需要的输出形态重新组织：

- 完整研究报告（所有结论 + 证据矩阵 + 风险点）
- 简报（核心判断 + 三档建议 + 关键证据，1–2 页）
- PPT 提纲（按 6 步法或按建议三档拆页）
- 监测追踪表（观察类建议的指标清单）

注意：再次强调，所有"建议"应该都打上"框架性候选 · 待研究员审核"标记，由研究员判断保留 / 修改 / 删除。

## 跨领域案例参考

不同研究类型的落地映射链长度和形态差异较大。读贴近课题的案例：

- `references/cases/case_token_outbound.md` —— 商业研究的长映射链（行业现象 → 推算 → 折算 → 服务对象承接），含"行动类故意留空"的样本（当前仓库唯一完整案例）

政策 / 技术路线类映射链案例暂未收录。普适纪律不变——**证据强度不到，行动类建议主动留空或降档**。

## 与报告成稿模块的衔接

如果用户要"出报告"、"成稿"、"按成熟体例写"，不要把本模块的"观察类 / 研究类 / 行动类"表格直接作为最终报告。下一步进入 `modules/07_report_assembly.md`，把分层建议改写为"一是、二是、三是"的报告语言，并按 `references/report_style_guide.md` 选择报告体例。

## 与课题管理类技能的衔接

走到第六步意味着综合研究主流程结束。后续动作通常是：

- **成果交付** —— 把分层建议转成完整报告 / 简报 / PPT / 监测表
- **课题归档** —— 把六步产出物结构化沉淀
- **观察类建议** —— 触发后续监测，可能升级为新课题

## 一句话总结

研究结论不等于对服务对象的建议。建议分层做的事是把"研究的发现"翻译成"服务对象能用的动作"：证据越强，动作越实；映射链走得越扎实，建议越站得住。


## 模块结束 · 硬停

<硬停>

**本模块到此结束。不要直接进入下一模块（成稿）或宣称交付完成，即使研究员回复"继续""下一步""好"。**

### 输出顺序

1. 按"阶段产出格式"输出本模块的阶段产出
2. 列出"待研究员逐条确认的判断点"——直接来自本模块"AI 不替你做的"小节，把每条转成针对当前课题的具体问题
3. 停下，等待研究员逐条回应

### 反惯性规则

- 研究员的简短确认（"好""继续""下一步""嗯""可以""go""next"）**不构成跳过本停止协议的授权**
- 研究员若只回应部分判断点，对未覆盖项继续追问，不要默认其余项已通过
- 确认完成之前，不要预先起草、预览模块 07 的报告内容

### 后续走向（由研究员选择）

研究员对所有判断点逐条表态，或明确说出"全部认可"后，按以下分支继续：

- 若需要面向领导/读者的成熟报告 → 进入 **模块 07 · 报告成稿组装**
- 若研究到此为止，仅交付分层建议 → 交付完成，不再继续
- 由研究员明确选择走向，不替研究员默认

</硬停>
