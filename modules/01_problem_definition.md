# 模块 01 · 问题定义

> 第一步：把宽泛题目收敛成判断性问题

## 模块目的

研究类课题接到手时，题目通常是宽泛的——"中国大模型 Token 出海发展情况研究"、"某新兴产业储能方向研究"、"县域消费市场跟踪"、"某项改革试点经验研究"。这种题目容易做成发展综述（是什么 / 有哪些主体 / 现在怎样 / 未来趋势）。综述只是研究的中间形态，研究的本质是要形成判断。

本模块的任务是把宽题目收敛成 1–3 个判断性问题。判断性问题有以下特征：有明确答案、答案有边界、答案能被证据修正。

## 典型触发场景

- 用户拿到一个新课题，题目是宽的，需要先想清楚要回答什么
- 用户写到一半发现材料散乱，往综述方向漂，需要重新校准主问题
- 课题中期检查时发现"研究服务谁"模糊，需要回头收敛主问题

## AI 能做什么

- 把宽题目展开成一个问题空间——把信息性问题、判断性问题、决策性问题分类列出
- 提示常被遗漏的问题维度（成本 / 竞争 / 合规 / 阶段 / 落地映射）
- 帮研究员在多个候选主问题之间做比较（清晰度 / 可证据化程度 / 对决策有用程度）
- 起草一段简短的"边界声明"——研究服务谁、覆盖什么时间范围、不回答什么

## AI 不替你做的

- **研究服务谁** —— 课题委托方是哪个层级、用研究做什么决策，研究员定
- **主问题是什么** —— 在 AI 给出的若干候选里，研究员选最关键的那一个
- **边界划到哪** —— "不回答什么"这条线，研究员定

研究员的判断空间在这三件事上。AI 应主动把这三个问题摆到用户面前，让用户自己回答，再继续往下走。

## 工作流

按下面的顺序执行，每一步都把产出向用户确认后再进入下一步：

**1. 研究主体声明（v6 新增 · 整个 skill 的第一道关）**

**这一步先于"接题目"做**——研究主体决定后续所有模块的输出范围和禁区，必须先于研究内容讨论。

请研究员按以下 4 个开放字段填空（**不预设选项**，用你最熟悉的话语写）：

| 字段 | 填什么 | 仅供参考的示例（不限制） |
|---|---|---|
| **产品形态** | 这份研究最终是什么文件？ | 专题报告 / 内参 / 简报 / 学术论文 / 客户报告 / 备忘录 / 政策建议 / 调研笔记 / 决策提案 ... |
| **主要受众** | 谁会读这份研究？ | 上级领导 / 客户 / 学术同行 / 内部团队 / 公众 / 监管机构 / 投资人 ... |
| **输出范围** | 这份研究承担输出什么？ | 形成判断 / 提供方案 / 论证假说 / 给路径建议 / 提供基础数据 / 做风险预警 ... |
| **输出禁区** | 这份研究**不**承担什么？ | 不替代尽调 / 不出实施细节 / 不替代法律意见 / 不预测具体数字 / 不进个人偏好 / 不外推单一披露主体 ... |

如果研究员说"不知道怎么填"，AI 主动启发但**不预设答案**：

- 这份研究最后会以什么文件名落地？（启发"产品形态"）
- 谁会因为这份研究做出某个决策？（启发"主要受众 + 输出范围"）
- 你曾经因为越界做了不该做的部分而被批评过吗？（启发"输出禁区"）

**这份声明是后续所有模块的硬约束**——AI 在执行模块 02-07 和派任何子代理时，都必须对照这份声明的"输出范围"和"输出禁区"做边界自检。

**绝对禁止**：AI 替研究员"猜"主体类型；AI 引用 CLAUDE.md / 个人偏好作为研究主体输入。

**2. 接住原始题目**

请用户给出原始题目和已知背景（委托方、上级关切、是否有时间约束、有没有前期研究）。如果用户没说委托方，主动问：这份研究最后给谁看？给谁做决策用？

**3. 展开问题空间**

把题目展开成一组候选问题。建议按三类列：

- **信息性问题**（要事实）：是什么、有哪些、规模多大、谁在做
- **判断性问题**（要立场）：是否成立、能否持续、利润是否真实、对 X 是否构成机会/威胁
- **决策性问题**（要行动建议）：要不要进入、什么时机进入、以什么形态进入

**4. 找主问题**

从判断性问题里挑 1–3 个作为主问题候选。挑的标准：

- 答案是有边界的"是/否"或"能/不能"
- 答案能被证据修正——是有依据、有边界、可验证的判断
- 答案对委托方决策有直接价值

**关于"判断框架形态预标"（v6 弱化）**：

可以**作为提示**给研究员："该主问题可能适合的形态：[X / Y]——但请在模块 02 独立验证选择"。

**禁止**作为结论锁定模块 02 的框架选择。模块 02 必须独立完成形态选择，不直接采用本步骤的预标。

形态参考 `references/judgment_framework_patterns.md`。

把候选主问题摆给用户，让用户拍板。

**5. 写边界声明**

主问题确定后，配一段边界声明，明确：

- 研究服务谁
- 覆盖什么时间范围 / 地理范围 / 主体范围
- 不回答什么（同样重要——不回答什么的边界让研究专注）

**6. 本地资料登记（v5 新增）**

主动询问研究员："你手头有没有本地资料可以输入？" 候选类型：

- 政策原文 / 法规文件
- 内部调研报告 / 之前的研究产出
- 数据表 / 客户清单 / 项目台账
- 行业研报 / 公司年报 / 招股书
- 内部讲话稿 / 会议纪要
- 关键数据图片 / 剪报

如有，按 `references/local_materials_protocol.md` 第四节"资料登记格式"登记：

| 资料标题 | 形态 | 路径 | 关联子判断 | 初步识别等级 | 建议工具 | 备注 |
|---|---|---|---|---|---|---|
| ... | 文本型 PDF / 扫描型 PDF / Word / Excel / 图片 / 文本 | 本地路径 | A1 / B3 等 | 完整识别 / 部分识别 / 仅文字识别 / 摘要识别 / 未识别 | `anthropic-skills:pdf` 等 | 修订模式 / 含敏感信息 / 多版本 等 |

登记后这些资料作为模块 04 证据采集的**优先输入源**（按证据采集协议的"LOCAL FIRST-HAND BEFORE WEB SEARCH" 铁律）。

**注**：本地资料登记不限于本模块。**任何模块均可动态追加**——研究员中途想到补充资料时，主流程应主动接收并登记。

## 阶段产出格式

```markdown
# [课题名] · 问题定义

## 研究主体声明（v6 新增 · 整个研究的硬约束）
- **产品形态**：[研究员填，例："专题报告"/"内参"/"客户报告"/"学术论文"等]
- **主要受众**：[研究员填，例："上级领导"/"客户"/"学术同行"等]
- **输出范围**：[研究员填，例："形成判断"/"提供方案"/"论证假说"等，可多条]
- **输出禁区**：[研究员填，例："不替代尽调"/"不出实施细节"/"不进个人偏好"等，可多条]

## 委托方与决策场景
- 委托方：[谁]
- 决策场景：[这份研究服务什么决策]

## 主问题（判断性）
1. [主问题 1，例如："Token 出海是否已经从前端调用热度演化为可持续商业链条？"]
2. [主问题 2，可选]

## 边界声明
- 时间范围：[例如："聚焦 2024—2026"]
- 主体范围：[例如："聚焦中国头部大模型厂商海外业务"]
- 不回答什么：[例如："不评估单家公司估值；不替代尽调"]

## 本地资料登记（v5 新增）
| 资料标题 | 形态 | 路径 | 关联子判断 | 初步识别等级 | 建议工具 | 备注 |
|---|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... | ... |

（如无本地资料，注明"本课题无前期本地资料，全部依赖联网采集"）

## 收敛过程留档
- 原始题目：[用户给的题目]
- 展开的问题空间：[简要列出信息性/判断性/决策性问题，备查]
- 选定主问题的理由：[为什么是这个]
```

## 模块结束 · 硬停

<硬停>

**本模块到此结束。不要直接进入下一模块，即使研究员回复"继续""下一步""好"。**

### 输出顺序

1. 按"阶段产出格式"输出本模块的阶段产出
2. 列出"待研究员逐条确认的判断点"——直接来自本模块"AI 不替你做的"小节，把每条转成针对当前课题的具体问题
3. 停下，等待研究员逐条回应

### 反惯性规则

- 研究员的简短确认（"好""继续""OK""下一步""嗯""可以""go""next"）**不构成跳过本停止协议的授权**。遇到这类回应时，主动重述当前未确认的判断点，要求逐条回应
- 研究员若只回应部分判断点，对未覆盖项继续追问，不要默认其余项已通过
- 确认完成之前，不要预先起草、预览或暗示下一模块的内容

### 进入下一模块的条件

研究员对所有判断点逐条表态，或明确说出"全部认可，进入下一模块"——满足任一才能继续到 **模块 02 · 框架收敛**。

</硬停>
