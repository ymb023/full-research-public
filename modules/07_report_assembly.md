# 模块 07 · 报告成稿组装

> 第七步：把六步研究产物转成成熟报告 + 完整交付包

## 铁律

```
NO DELIVERY WITHOUT VERIFICATION CHECKLIST
```

成稿交付时**必须配套生成"待复核清单"**——汇总模块 04/05/06 中所有需要研究员人工核对的位置（识别等级非完整、边界待确认、单一信源、必改风险、▢ 占位符未填等）。无待复核清单的成稿不算完整交付。

详见本模块"交付包结构"小节。

## 模块目的

六步法产出的是研究过程和判断材料，成熟报告呈现的是面向领导和决策的结论、证据和建议。本模块负责隐藏方法论脚手架，把问题定义、判断框架、假设、证据矩阵、反方审稿和分层建议组装成符合惯常体例的报告。

## 典型触发场景

- 用户说"出报告"、"写成成熟报告"、"按惯常体例成稿"、"给领导看的稿子"
- 六步流程完成后，需要把阶段产物转成最终报告
- 用户给了参考报告或示例，希望按类似体例输出
- 现有草稿像过程稿、综述稿或方法论稿，需要改成正式研究报告

## 必读资源

- `references/report_style_guide.md` —— 成熟报告体例、段落写法、标题层级、建议措辞
- `assets/mature_report_template.md` —— 两类最终报告模板

如用户提供新的参考报告，先从参考中抽取体例，再与 `report_style_guide.md` 合并使用。

## AI 能做什么

- 判断应使用"机构专题报告"还是"领导汇报式分析报告"外壳（体例可换成你单位的）
- 把六步产物映射为报告章节，而不是逐项照搬
- 起草核心观点摘要、正文和相关建议
- 将"观察 / 研究 / 行动"三档建议改写为成熟报告中的"一是、二是、三是"
- 把证据矩阵、案例对照、测算表等放入附件或表格位置
- 检查标题、段落、建议和证据强度是否符合成熟体例

## AI 不替你做的

- **最终报送口径** —— 是否可直接报领导，由研究员和部门把关
- **内部数据真实性** —— 资产、场站、收益、组织分工等内部数据需要研究员核对
- **政治分寸和敏感边界** —— 涉及决策层、监管态度、外部主体评价时需要人工审慎
- **签发信息** —— 批准、校核、编写、期号、分送范围等由用户确认

## 工作流

> **v6 主体声明自检**：执行本模块前，必须对照模块 01 的"研究主体声明"四字段（产品形态 / 主要受众 / 输出范围 / 输出禁区）做边界自检——任何动作若超出"输出范围"或落入"输出禁区"，立即停下并降档。


**1. 选择报告外壳**

按交付对象选择：

- 给领导、部门或专题汇报：用"领导汇报式分析报告"
- 作为机构正式报告、专题研究或知识产品：用"机构专题报告"

**优先用户自己的体例**：如果用户提供了本单位的参考报告样例，先从样例抽取体例，模板只作兜底。`assets/mature_report_template.md` 里的两个外壳是通用示例，不是强制格式。

不确定且用户没提供样例时，先给用户两个外壳的差别；若用户不选，默认机构专题报告外壳。

**2. 把六步产物映射为读者章节**

不要把六步名写成最终标题。按下表转换：

| 六步产物 | 报告中呈现为 |
|---|---|
| 问题定义 | 标题、开头缘起、研究边界的隐含约束 |
| 判断框架 | 正文章节结构 |
| 假设清单 | 章节中的待验证判断，不单独暴露 |
| 证据矩阵 | 数据、案例、图表、附件 |
| 反方审稿 | 谨慎措辞、风险分析、边界说明 |
| 分层建议 | "相关建议"或"下一步建议" |

**3. 先写核心观点或开头**

机构专题报告必须有 `【核心观点摘要】`。领导汇报式报告用报送对象和"报告如下："开头。开头必须先给判断和意义，不要先讲研究过程。

**4. 写正文**

每个一级章节围绕一个主判断。每个自然段采用：

```text
判断句。关键证据/数据/案例。对服务对象或下一判断的意义。
```

章节之间要递进：外部趋势 / 机制 → 国内实践或对标 → 服务对象适配 / 风险 → 建议。

**5. 改写建议**

把模块 06 的三档建议翻译为成熟报告动作语言：

- 观察类 → "持续跟踪", "建立监测口径", "保持前瞻性关注"
- 研究类 → "开展专题研究", "摸排资源", "形成评估框架", "建立工作机制"
- 行动类 → "启动试点", "加快布局", "推动实施", "完善管理机制"

最终报告中用"一是、二是、三是"组织，不把"观察类/研究类/行动类"作为标题。

**6. 安排附件**

详细证据矩阵、项目清单、场站清单、案例对照、测算过程放入附件。正文只保留支撑判断所需的关键数据。

**7. 生成交付包（v5 新增 · 铁律强制）**

按下方"交付包结构"小节生成完整交付包。**核心是"待复核清单"+"跨模块待研究员补汇总"两份新增产出**——这两份和成熟报告一起作为附件交付。

**8. 成稿自检**

交付前逐项检查：

- 开头是否直接给出核心判断
- 标题是否是读者语言，而不是方法论语言
- 正文是否每节先判断、再证据、再含义
- 建议是否落到服务对象能采取的动作
- 证据不足的地方是否降档或标注边界
- 过程性材料是否进入附件而非正文
- **（v5 新增）待复核清单是否生成且非空（如真空，标注"全部复核已完成"）**
- **（v5 新增）跨模块待补汇总是否生成**

## 交付包结构（v5 新增）

完整交付包采用**单一报告 + 多个附件**形态，分三层：

### A · 对外交付（给领导 / 委托方 / 外部读者）

成熟报告（隐藏脚手架）+ 关键附件：

| 内容 | 说明 |
|---|---|
| 报告正文 | 机构专题报告或领导汇报式分析报告（按 `references/report_style_guide.md`）|
| 附件 1：核心数据表 | 正文支撑数据汇总 |
| 附件 2：政策清单 | 引用的政策文件原文索引 |
| 附件 3：典型项目对标表 | 关键案例对照 |

### B · 对内交付（给研究员 / 课题组 / 上级核查）

完整研究档案：

| 内容 | 来源 | 说明 |
|---|---|---|
| 本地资料登记表 | 模块 01 第 5 步 | 全部本地资料的登记 |
| 证据矩阵（12 字段） | 模块 04 | 通过协议采集的"完整识别"证据条目 |
| **待复核清单** | **v5 新增 · 自动汇总** | 见下方 |
| **跨模块待研究员补汇总** | **v5 新增 · 自动汇总** | 见下方 |
| 假设清单 | 模块 03 | 含正向 / 反向 / 先验置信度 |
| 反方审稿报告 | 模块 05 | 必改 / 建议改 / 可保留三档风险 |
| 分层建议（含落地映射链） | 模块 06 | 含 ▢ 占位符的映射链 |

### C · 课题归档（备查 / 复盘 / 审计）

全过程留档：

| 内容 | 用途 |
|---|---|
| 六步法每步原始产出 | 复盘方法论执行质量 |
| 所有子代理报告 | 审查确认偏差排查情况 |
| 边界拒绝清单 | 证明研究边界被严守 |
| 证据缺口清单 | 下一轮 v2 研究的立项依据 |

## 「待复核清单」自动生成规则（v5 新增）

汇总以下来源到一张可勾选的 markdown 表：

| 来源模块 | 触发条件 | 进入清单的字段 |
|---|---|---|
| 模块 04 证据矩阵 | 识别等级 ≠ "完整识别" | 资料标题 / 识别等级 / 复核动作（回原文核对） |
| 模块 04 证据矩阵 | 标 `[待边界确认]` | 证据条目 / 边界冲突点 / 裁决动作 |
| 模块 04 证据矩阵 | 单一信源支撑关键判断 | 证据条目 / 多源补查方向 |
| 模块 05 反方审稿 | "必改"风险 | 风险点 / 修改动作 |
| 模块 05 反方审稿 | "建议改"风险 | 风险点 / 修改动作 |
| 模块 06 落地映射链 | ▢ 占位符未填 | 链中环节 / 待研究员补的"为什么成立"理由 |
| 模块 06 启发性问题 | "由谁牵头" / "时间或资源" 未答 | 问题 / 待研究员答 |

输出形态：

```markdown
# 待复核清单（汇总自模块 04/05/06）

## 必改类（不改影响结论成立）
- [ ] [来源] [具体条目] → [修改动作]
- [ ] ...

## 待复核（识别等级 / 边界确认 / 单源补强）
- [ ] [来源] [具体条目] → [复核动作]
- [ ] ...

## 待研究员答（启发性问题 / 映射链 ▢）
- [ ] [来源] [问题] → [待答内容]
- [ ] ...

—— 总计 N 项，建议研究员逐项打勾后再正式报送
```

**规则**：报告交付时如本清单非空，主流程必须显式提示研究员"该报告含 N 项待复核内容，建议在报送前完成"。

## 「跨模块待研究员补」汇总自动生成规则（v5 新增）

把六步法所有"AI 不替你做的"未答项 + 缺口集中到一张表，作为：

- 当前研究的"已知边界"（领导/委托方知情用）
- v2 研究的立项依据

汇总来源：

| 模块 | 待补内容 |
|---|---|
| 模块 01 | 委托方 / 主问题 / 边界——研究员未确认项 |
| 模块 02 | 框架与主问题对应 / 模块重叠 / 委托方关切对齐——未确认项 |
| 模块 03 | 先验置信度 / 优先验证假设——未拍板项 |
| 模块 04 | 证据等级判断 / "不能支撑什么" 边界——未审核项 |
| 模块 05 | 风险分级 / 修改建议——未决项 |
| 模块 06 | 落地映射链 / 启发性问题——研究员未答项 |

输出形态：

```markdown
# 跨模块待研究员补汇总

## 关键缺口（影响主结论）
| 模块 | 待补内容 | 优先级 | 备注 |
|---|---|---|---|
| ... | ... | 高/中/低 | ... |

## 次级缺口（影响严谨度）
...

## 外延缺口（已在边界排除，仅供参考）
...
```

## 阶段产出格式

使用 `assets/mature_report_template.md`，输出一份完整报告草稿。若用户只要提纲，输出到二级标题和每节核心判断即可。

**v5 强制配套**：报告草稿 + 附件 4「待复核清单」+ 附件 5「跨模块待研究员补汇总」。三份一起作为完整交付包。
