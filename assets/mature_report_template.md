# 成熟研究报告模板

二选一形态。交付前删掉未用的那一份。

> **这两个外壳是通用示例**。如果你所在机构有自己的报告体例（红头格式、固定栏目、签发栏），优先用你单位的样例，本模板只作兜底参考。模板里的"服务对象 / 机构名"等占位字样，按你实际情况替换。

## 形态 A：机构专题报告

```markdown
# [报告标题]

（[机构名][年份]年报告 第[期]期）

【核心观点摘要】

[用 3-6 个短段落写清楚核心判断、对服务对象的意义和建议。不要出现"问题定义/证据矩阵/反方审稿"等方法论词。]

【正文】

一、[外部趋势 / 机制 / 形势判断]

[判断句。关键数据或案例。对下一步分析的意义。]

（一）[子判断标题]

[判断句。证据。边界或影响。]

（二）[子判断标题]

[判断句。证据。边界或影响。]

二、[国内实践 / 行业路径 / 对标案例]

[按判断维度组织，不按材料来源堆叠。]

三、[服务对象适配性 / 影响 / 风险分析]

[把外部结论映射到服务对象的资源、能力、战略、风险或时机。]

四、相关建议

一是[建议动作]。[依据与目标。执行牵头或研究抓手。]

二是[建议动作]。[依据与目标。执行牵头或研究抓手。]

三是[建议动作]。[依据与目标。执行牵头或研究抓手。]

批准：[姓名]

校核：[姓名/部门]

编写：[姓名/部门]

附件

附件1 [数据表 / 清单 / 案例对照表]
```

## 形态 B：领导汇报式分析报告

```markdown
# 关于[主题]的分析报告

[报送对象]：

[牵头部门/单位]围绕[任务来源/研究事项]，深入调研[对象]，梳理分析[关键内容]，报告如下：

一、[认识与形势分析]

[判断句。关键政策/趋势/数据。对服务对象或决策场景的意义。]

二、[现状基础 / 既有实践]

[服务对象已经做了什么，形成了什么基础，仍有什么短板。]

三、[典型模式 / 重点问题 / 研判分析]

（一）[模式或问题一]

[判断句。案例或数据。适用边界。]

（二）[模式或问题二]

[判断句。案例或数据。适用边界。]

四、下一步建议

[过渡段：结合服务对象定位、任务要求和研究判断，提出下一步安排。]

一是[建议动作]。[如何做、谁来做、时间或机制。]

二是[建议动作]。[如何做、谁来做、时间或机制。]

三是[建议动作]。[如何做、谁来做、时间或机制。]

[部门/单位]

[日期]

附件1 [必要的数据表或模式总结]
```
